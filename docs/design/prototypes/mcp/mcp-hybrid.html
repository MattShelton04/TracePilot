📁 Config File Locations
MCP server configurations are stored as JSON. TracePilot reads from multiple locations with a merge strategy (project overrides global).
// Global config (all projects) ~/.config/tracepilot/mcp-servers.json // Project-specific config <project-root>/.tracepilot/mcp-servers.json // VS Code compatibility (read-only import) <project-root>/.vscode/mcp.json
  • Global config is created on first launch
  • Project configs are optional and override global settings
  • VS Code configs can be imported via the "Import from VS Code" action
🦀 Rust Types
Core types for MCP server configuration in the Tauri backend.
use std::collections::HashMap; use serde::{Serialize, Deserialize}; /// Transport type for MCP communication #[derive(Serialize, Deserialize, Clone)] enum McpTransport { Stdio { command: String, args: Vec<String> }, Http { url: String }, Sse { url: String }, } /// Configuration for a single MCP server #[derive(Serialize, Deserialize, Clone)] struct McpServerConfig { name: String, transport: McpTransport, env: HashMap<String, String>, scope: McpScope, enabled_tools: Option<Vec<String>>, status: McpServerStatus, } #[derive(Serialize, Deserialize, Clone)] enum McpScope { Global, Project } #[derive(Serialize, Deserialize, Clone)] enum McpServerStatus { Active, Paused, Error(String) }
⚡ IPC Commands (Tauri)
All MCP operations use Tauri's IPC command system. Commands are async and return Result types.
// List all configured servers #[tauri::command] async fn mcp_list_servers() -> Result<Vec<McpServer>> // Add a new server configuration #[tauri::command] async fn mcp_add_server(config: McpServerConfig) -> Result<McpServer> // Remove a server by ID #[tauri::command] async fn mcp_remove_server(id: String) -> Result<()> // Test connectivity to a server #[tauri::command] async fn mcp_test_connection(id: String) -> Result<ConnectionResult> // Toggle a specific tool on/off #[tauri::command] async fn mcp_toggle_tool(server_id: String, tool: String, enabled: bool) -> Result<()> // Update server config (triggers diff preview) #[tauri::command] async fn mcp_update_server(id: String, config: McpServerConfig) -> Result<()>
🔄 Test Connection Flow
The test connection button triggers a multi-step validation sequence.
  • 1. Spawn process — For stdio: spawn command with args. For http/sse: open connection to URL.
  • 2. Protocol handshake — Send initialize request, validate server capabilities and protocol version.
  • 3. List tools — Call tools/list to discover available tools. Compare with cached tool list.
  • 4. Report status — Return response time, tool count, protocol version. Update health stats.
  • 5. Teardown — Close connection gracefully. For stdio: send SIGTERM then SIGKILL after 5s timeout.
🔧 Enable/Disable Strategy
Tool toggling is implemented at the proxy layer. Disabled tools are never forwarded to the LLM context, reducing token consumption.
// In the MCP proxy middleware: fn filter_tools(tools: Vec<Tool>, config: &McpServerConfig) -> Vec<Tool> { match &config.enabled_tools { Some(allowed) => tools.into_iter() .filter(|t| allowed.contains(&t.name)) .collect(), None => tools, // All tools enabled by default } }
  • Default: all tools enabled (enabled_tools: None)
  • Toggling stores explicit list in config file
  • Token savings are estimated at ~4 chars per token from tool schemas
  • Changes take effect on next LLM interaction (no server restart needed)
💾 VS Code Import
TracePilot can import MCP server configs from VS Code's .vscode/mcp.json format.
// VS Code format (.vscode/mcp.json) { "mcpServers": { "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_TOKEN": "${env:GITHUB_TOKEN}" } } } } // Import resolves ${env:VAR} references and merges // into TracePilot's config with conflict detection
🏪 Pinia Store Pattern
Frontend state is managed via a Pinia store with typed state, computed getters, and IPC-backed actions.
// stores/mcpServers.ts export const useServerStore = defineStore('mcpServers', () => { const servers = ref<McpServer[]>([]); const loading = ref(false); // Computed const activeServers = computed(() => servers.value.filter(s => s.status === 'active') ); const totalTokens = computed(() => activeServers.value.reduce((sum, s) => sum + s.tools.filter(t => t.enabled) .reduce((ts, t) => ts + t.estimatedTokens, 0), 0) ); // Actions async function fetchServers() { loading.value = true; servers.value = await invoke('mcp_list_servers'); loading.value = false; } async function toggleTool(serverId: string, tool: string) { await invoke('mcp_toggle_tool', { serverId, tool }); await fetchServers(); } return { servers, loading, activeServers, totalTokens, fetchServers, toggleTool }; });
