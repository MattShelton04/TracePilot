Session Summary
Built-in preset · v1
Description

Generate a concise, structured summary of a Copilot session. Captures key decisions made during the session, all files that were created or modified, tool calls executed, and provides an overall outcome assessment with confidence scoring.

Prompt Template
● System Prompt
You are a session analysis assistant for TracePilot. Your role is to produce concise, actionable summaries of Copilot coding sessions. Always structure output using the provided schema. Be factual and precise. Include confidence scores where applicable.
● User Prompt
Analyse the following session data and produce a structured summary. Session ID: {{session_id}} Duration: {{session_duration}} Turn count: {{turn_count}} Session transcript: {{session_transcript}} File changes: {{file_changes}} Produce a summary covering: 1. Executive overview (2-3 sentences) 2. Key decisions with rationale 3. Files modified with change description 4. Tool usage patterns 5. Outcome assessment with confidence score
Variable Definitions
Variable Type Source
session_id string Session metadata
session_duration string Computed from timestamps
turn_count number Session turn array length
session_transcript string Formatted turn content
file_changes string Tool call file operations
Context Sources
📄
Session Metadata id, timestamps, model
💬
Turn History all turns, max 200
🔧
Tool Calls name, args, result summary
Output Schema
{ "summary": { "executive_overview": string, "key_decisions": [ { "decision": string, "rationale": string, "confidence": number } ], "files_modified": [ { "path": string, "change_type": "created" | "modified" | "deleted", "description": string } ], "outcome_assessment": { "status": "success" | "partial" | "failed", "confidence": number, "notes": string } } }
