Playwright CLI Skill

Overview

This skill automates browser interactions for web testing, form filling, screenshots, and data extraction using the Playwright framework via the CLI.

Capabilities

  • Navigate to URLs and interact with web pages
  • Fill forms and submit data automatically
  • Take screenshots of full pages or specific elements
  • Extract structured data from web pages
  • Test web applications end-to-end
  • Handle authentication flows and cookies

Usage Guidelines

  1. Always verify URLs before navigating
  2. Use headless mode by default for efficiency
  3. Take screenshots as evidence of completed actions
  4. Handle errors gracefully — retry on transient failures
  5. Respect rate limits and add delays between requests

Security Notes

  • This skill has allowed-tools: Bash(playwright-cli:*) access
  • Only execute playwright-cli commands, never arbitrary shell
  • Do not store or transmit credentials found on pages

SKILL.md Format

Each skill is a single markdown file with YAML frontmatter:

---
name: playwright-cli
description: "Automates browser interactions for testing"
allowed-tools: Bash(playwright-cli:*)
scope: global
license: MIT
compatibility: "copilot-cli >= 0.4.0"
---

# Skill instructions in markdown...
Body content becomes the system prompt context.

Frontmatter Fields

  • name — Unique identifier (kebab-case)
  • description — Human-readable summary (max 1024 chars)
  • allowed-tools — Tool access grants (security-sensitive)
  • scope — global or project
  • license — SPDX license identifier
  • compatibility — Version constraint

Rust Types

pub struct Skill {
    pub name: String,
    pub description: String,
    pub scope: SkillScope,
    pub allowed_tools: Option<Vec<String>>,
    pub enabled: bool,
    pub body: String,
    pub assets: Vec<SkillAsset>,
    pub token_estimate: usize,
}

pub enum SkillScope {
    Global,  // ~/.copilot/skills/
    Project, // .github/skills/
}

GitHub Private Repo Import

Two authentication methods for importing from private repositories:

1. gh CLI (Recommended)

Uses gh auth status to check authentication, then gh api to fetch repo contents. Preferred because it inherits existing terminal auth.

2. Personal Access Token

Fallback: user provides a PAT with repo scope. Stored encrypted in OS keychain via keyring crate.

Enable/Disable Strategy

Skills are disabled by renaming SKILL.md to SKILL.md.disabled. This is simple, reversible, and works with git:

# Disable
mv ~/.copilot/skills/code-review/SKILL.md \
   ~/.copilot/skills/code-review/SKILL.md.disabled

# Enable
mv ~/.copilot/skills/code-review/SKILL.md.disabled \
   ~/.copilot/skills/code-review/SKILL.md

Token Estimation

Approximation: ~4 characters per token. The full SKILL.md content (frontmatter + body) is included in the context window. Token budget tracking helps users understand the cost of having many skills active.
