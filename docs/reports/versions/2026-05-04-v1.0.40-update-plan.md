# Copilot CLI v1.0.24 to v1.0.40 Update Plan

## Problem

TracePilot's Copilot CLI schema support was last refreshed around v1.0.24, while the installed CLI is now v1.0.40. The event schema, RPC schema, and persisted session corpus have moved enough that TracePilot should add typed support for newly persisted events, preserve older sessions, and improve the version-analysis tooling so future refreshes are less manual.

## Evidence gathered

- Existing command: `pnpm cli versions diff 1.0.24 1.0.40 --json`.
- Installed package versions under `C:\Users\mattt\.copilot\pkg\universal`: `0.0.410`, `1.0.24`, `1.0.35`, `1.0.36`, `1.0.37`, `1.0.39`, `1.0.40`.
- Local session corpus scanned from `C:\Users\mattt\.copilot\session-state`: 348 sessions, about 755k JSONL events, no malformed event lines in the schema scan.
- The v1.0.40 schema has 78 event variants: 39 persisted/optional events and 39 always-ephemeral events.
- Current coverage before this work: 36 of 39 persisted schema events typed/handled by TracePilot (92.3%).

## v1.0.24 to v1.0.40 schema changes that matter

### New event types

| Event | Persisted? | Observed locally? | Value |
| --- | --- | --- | --- |
| `permission.requested` | Optional/persisted | Yes, 4 events in 2 v1.0.40 sessions | Auditable approval prompts, shell/write/read/MCP/custom-tool intent, session/location approval options. |
| `permission.completed` | Optional/persisted | Yes, 4 events in 2 v1.0.40 sessions | Approval/denial outcome, feedback, cancellation/rejection reason, tool-call correlation. |
| `external_tool.requested` | Optional/persisted | Not yet | Future external-tool handoff visibility with trace context. |
| `assistant.message_start` | Always ephemeral | No | Live-stream UI could show phase/message boundaries earlier than completed messages. |
| `model.call_failure` | Always ephemeral | No | Live-stream reliability telemetry and auto-switch diagnosis. |
| `auto_mode_switch.requested` | Always ephemeral | No | Live-stream model fallback/auto-switch request visibility. |
| `auto_mode_switch.completed` | Always ephemeral | No | Live-stream model fallback/auto-switch result visibility. |

`session.import_legacy` was removed from the v1.0.40 schema, but TracePilot should keep parsing it for older session support.

### Persisted event field changes

- `permission.requested` now contains both legacy-ish `permissionRequest` and newer `promptRequest` shapes in observed v1.0.40 sessions. Observed kinds include shell/commands and write prompts.
- `permission.completed` contains `requestId`, `toolCallId`, and a union-style `result.kind`, including interactive rejection feedback in observed data.
- `session.compaction_complete.compactionTokensUsed` changed from `{ input, output, cachedInput }` to `{ inputTokens, outputTokens, cacheReadTokens, cacheWriteTokens, duration, model, copilotUsage }`. Local data contains both old and new shapes, so parsing must support both.
- `session.error` added `errorCode` and `eligibleForAutoSwitch`; local data has at least one event with both.
- `session.model_change` added `cause`; local data has at least one.
- `session.resume` added `sessionWasActive` and `continuePendingWork`.
- `session.context_changed` / start/resume context added `repositoryHost`.
- `session.shutdown` added `totalNanoAiu` and `tokenDetails` in the schema. The current local persisted shutdown events do not yet show those fields, but the summary model should accept them.
- `assistant.message`, `tool.execution_start`, and `tool.execution_complete` added `turnId`; local data contains these fields in v1.0.40 events and the turn reconstructor should use them where useful.
- `user.message` added native document MIME/fallback fields and `parentAgentTaskId`; local data contains these fields.
- `system.notification.kind` gained `instruction_discovered` and `new_inbox_message`.

## Implementation plan

1. Add typed Rust model support for the persisted v1.0.40 gaps: `permission.requested`, `permission.completed`, and `external_tool.requested`.
2. Preserve older sessions by retaining removed/legacy event parsing (`session.import_legacy`) and by accepting both old and new field names for compaction token usage.
3. Extend existing data structs for modified persisted schemas: session context `repositoryHost`, resume flags, error auto-switch fields, info tip, model-change cause, shutdown AIU/token details, user-message native-document fields, and `turnId` on assistant/tool events.
4. Surface permission lifecycle events in reconstructed turns as low-risk session events with concise summaries. This makes approvals/denials visible in the conversation timeline without adding a large new UI surface.
5. Extend TypeScript shared event/payload types and known-event coverage to include the newly typed persisted events.
6. Improve the `versions` CLI tooling:
   - Add report metadata for a selected baseline/latest diff.
   - Include empirical observations for observed fields and unknown observed events.
   - Explicitly separate persisted gaps from always-ephemeral live-stream opportunities so coverage does not imply TracePilot should parse events that never land in `events.jsonl`.
   - Make report generation more automation-friendly for the cleanup-prone Copilot package cache by showing installed-version caveats and the exact diff window.
7. Generate a fresh report under `docs\reports\versions` after implementation.
8. Add focused tests for parsing, turn surfacing, and version-report output, then run the existing relevant Rust/TypeScript validation.
9. Run a final `claude-opus-4.7` review subagent, validate each finding, and apply relevant fixes.

## UI/UX surfacing

The lowest-risk UI improvement is to reuse the existing per-turn session-event surface. Permission events are high-value because they answer: "What did Copilot ask permission to do, and what happened?" Surfacing them as warning/info timeline chips gives users auditability without requiring a new approval-history page.

Further UI opportunities to document in the generated report:

- Permission audit trail: filterable approval/denial history by tool call, prompt kind, and result.
- Model reliability lane: ephemeral `model.call_failure` and `auto_mode_switch.*` events could enrich live-session monitoring when SDK/live-stream ingestion is available.
- Cost/usage precision: shutdown `totalNanoAiu`, `tokenDetails`, and updated compaction token usage can power richer per-model and per-compaction cost/tokens views.
- Native-document context: `supportedNativeDocumentMimeTypes` and fallback paths could explain document attachment handling.
- External-tool handoff visibility: `external_tool.requested` can expose trace context and tool names once observed.

## Expected value adds

The biggest immediate value add is permission observability: users can see when Copilot requested shell/write/read/MCP/custom-tool approval and whether it was approved, rejected, cancelled, or denied by rules/hooks. The second is better token/cost fidelity from the new shutdown and compaction usage fields. The third is future live-session UX: the ephemeral model-failure and auto-mode-switch events can explain why a model changed or a response retried, even though they are not persisted today.
