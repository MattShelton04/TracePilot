## 1. Executive Summary

Between Copilot CLI v1.0.7 and v1.0.24, there have been **61 changes**: 9 new event types, 23 modified events with new fields, 23 new RPC methods, and 1 new agent (`rubber-duck`). TracePilot currently handles **36/37 persisted event types (97.3% coverage)** — only `session.remote_steerable_changed` is unhandled.

However, **the bigger gap is not event types but missing fields on existing structs**. Multiple high-value fields are being silently dropped by serde's `Option<T>` deserialization:

| Category | Count | Impact |
|----------|-------|--------|
| Unhandled event types | 1 | Low — `session.remote_steerable_changed` not yet observed locally |

| New RPC methods | 23 | Informational — sessionFs, MCP config, history management |
| New agents | 1 | Low — `rubber-duck` agent added in v1.0.19 |

### Key Findings

1. **Token breakdown fields (`systemTokens`, `conversationTokens`, `toolDefinitionsTokens`) are present in real sessions but NOT captured** by `ShutdownData`, `CompactionStartData`, or `CompactionCompleteData` structs.
2. **Subagent performance metrics (`model`, `totalToolCalls`, `totalTokens`, `durationMs`) are in every v1.0.24 session** but `SubagentCompletedData` and `SubagentFailedData` don't have these fields.
3. **`assistant.message.requestId`** is present on every assistant message in v1.0.24 sessions — useful for tracing LLM requests.
4. **`UsageMetrics` struct is missing `reasoningTokens`** — visible in `session.shutdown` model metrics data.
5. **`assistant.usage` is ephemeral** (never persisted to disk), but the diff shows new fields (`reasoningTokens`, `ttftMs`, `interTokenLatencyMs`) that could power real-time analytics if we ever capture them.

## 7. Backward Compatibility Strategy

### Current Robustness (Already Excellent)

1. **`Option<T>` everywhere** — New fields on existing structs are silently ignored (None). Adding them is backward-compatible.
2. **`Unknown(String)` catch-all** — New event types don't crash parsing.
3. **`Other(Value)` fallback** — Failed typed deserialization falls back gracefully.
4. **`serde(rename_all = "camelCase")`** — Field naming is consistent.

### Recommendations for Maintaining Compatibility

1. **All new fields MUST be `Option<T>`** — Never add required fields to existing structs.
2. **Never remove enum variants** — Old sessions may contain deprecated event types.
3. **Use `#[serde(default)]`** on struct fields that changed type — e.g., if a field goes from `Option<String>` to `Option<Value>`, the wider type (`Value`) should be used.
4. **Version-aware analytics** — When computing analytics that depend on new fields, check the session's `copilotVersion` to know what data is available. Don't show "0 reasoning tokens" for sessions from before reasoning was tracked — show "N/A" instead.
5. **Index DB migration** — If new fields are added to the index, use `ALTER TABLE ... ADD COLUMN` with `DEFAULT NULL` (already the pattern used in TracePilot migrations).

## 9. Testing Strategy for Version Parsing

### Proposed Approach: Local Session Regression Test Suite

The idea is to create a test fixture directory containing representative `events.jsonl` files from each CLI version, then run parsing tests against them.

#### Step 1: Build Test Fixtures

```powershell
# Script to extract minimal test fixtures from real sessions
# For each CLI version with local sessions, extract:
#   - session.start (version identifier)
#   - One of each event type observed
#   - session.shutdown (metrics)

$sessionDir = "$env:USERPROFILE\.copilot\session-state"
$fixtureDir = "crates\tracepilot-core\tests\fixtures\versions"

# Group sessions by copilot version
# Extract representative events into fixture JSONL files
```

#### Step 2: Rust Integration Tests

```rust
// crates/tracepilot-core/tests/version_compat.rs
use std::path::{Path, PathBuf};
use tracepilot_core::parsing::events::parse_typed_events;
use tracepilot_core::models::SessionEventType;

fn fixtures_dir() -> PathBuf {
    Path::new(env!("CARGO_MANIFEST_DIR")).join("tests/fixtures/versions")
}

/// Test that every fixture file from every CLI version parses without errors.
#[test]
fn all_version_fixtures_parse_cleanly() {
    let dir = fixtures_dir();
    
    for entry in std::fs::read_dir(&dir).unwrap() {
        let path = entry.unwrap().path();
        if path.extension() == Some(std::ffi::OsStr::new("jsonl")) {
            let result = parse_typed_events(&path)
                .expect(&format!("Failed to parse {}", path.display()));
            // Reject Unknown event types (all persisted types should be recognized)
            for event in &result.events {
                assert!(!matches!(event.event_type, SessionEventType::Unknown(_)),
                    "Unknown event type in {}: {:?}", 
                    path.display(), event.event_type);
            }
            // Note: DeserializationFailed warnings are acceptable (schema fallback)
        }
    }
}

/// Test that new fields are correctly deserialized from v1.0.24 sessions.
#[test]
fn v1_0_24_new_fields_captured() {
    let path = fixtures_dir().join("v1.0.24-sample.jsonl");
    let result = parse_typed_events(&path)
        .expect("Failed to parse v1.0.24 fixture");
    
    // Find shutdown event and verify token breakdown
    let shutdown = result.events.iter()
        .find(|e| matches!(e.event_type, SessionEventType::SessionShutdown))
        .expect("Missing shutdown event");
    
    if let TypedEventData::Shutdown(data) = &shutdown.typed_data {
        assert!(data.system_tokens.is_some(), "Missing systemTokens");
        assert!(data.conversation_tokens.is_some(), "Missing conversationTokens");
    }
}

/// Test backward compatibility: old sessions still parse correctly.
#[test]
fn v1_0_5_still_parses() {
    let path = fixtures_dir().join("v1.0.5-sample.jsonl");
    let result = parse_typed_events(&path)
        .expect("Failed to parse v1.0.5 fixture");
    // New fields should be None
    let shutdown = result.events.iter()
        .find(|e| matches!(e.event_type, SessionEventType::SessionShutdown));
    if let Some(ev) = shutdown {
        if let TypedEventData::Shutdown(data) = &ev.typed_data {
            assert!(data.system_tokens.is_none(), "Old sessions shouldn't have systemTokens");
        }
    }
}
```

> ⚠️ **Privacy:** Fixture files MUST be synthetic — do NOT commit real session data. Real sessions contain user prompts, code snippets, file paths, and correlatable session IDs. Either:
> - (a) Manually author synthetic fixture JSONL with fictional content and random UUIDs, or
> - (b) Build a sanitization script that replaces paths/content/IDs before committing
>
> The existing `crates/tracepilot-core/src/testing.rs` helpers (lines 38–83) provide synthetic event builders that should be extended for this purpose.

> ⚠️ **Existing tests:** The `events.rs` test suite (lines 696–989) includes `sample_jsonl()` and `resumed_session_jsonl()` fixtures. These must also be extended with the new fields to ensure `combine_shutdown_data()` and `combine_model_metrics()` handle them correctly.
