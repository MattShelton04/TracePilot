# ADR-0002: IPC contract via Specta-generated bindings + invoke wrappers

Date: 2026-04-22
Status: Accepted

## Context

The Tauri webview talks to the Rust backend over two channels:

1. **Commands** (`invoke("plugin:tracepilot|<name>", args)`) —
   request/response RPC.
2. **Events** (`listen("<name>", cb)`) — backend-pushed notifications.

Prior to Specta adoption, TypeScript types on the webview side were
hand-maintained mirrors of Rust structs. Every change to a Rust model
required a corresponding manual `.ts` edit. Drift was frequent and
subtle: a field renamed in Rust would compile on both sides until a
runtime `undefined` surfaced in the UI. Wire-format changes (e.g.
`#[serde(rename_all = "camelCase")]` vs snake_case) were particularly
error-prone.

A second issue was the command name registry. Rust registers commands
by `&str`; TypeScript callers also refer to them by string. A typo on
either side compiled cleanly and failed only at runtime.

## Decision

TracePilot commits to a **generator-as-source-of-truth** IPC contract.

1. Every IPC-exposed Rust type derives `specta::Type`. A
   `cargo run --bin gen-bindings` binary in
   `tracepilot-tauri-bindings` walks the allow-list of commands and
   types and emits:
   - `packages/client/src/generated/bindings.ts` — typed `commands.*`
     object, event type mirrors, the `BindingsErrorIpc` envelope.
   - `packages/client/src/generated/ipc-commands.json` — sorted, stable
     manifest of every command name.
   The generated header contains `DO NOT EDIT — generated by
   pnpm gen:bindings`.
2. Hand-written IPC callers live in `packages/client/src/*.ts`
   (`sessions.ts`, `tasks.ts`, `sdk.ts`, etc.) and import **only** from
   the generated bindings. The typed `commands.*` helpers return
   `Result`-shaped promises via `typedError<T, BindingsErrorIpc>`
   (see the generated file header).
3. The invoke shim (`packages/client/src/invoke.ts`) wraps every call
   with performance instrumentation (slow-call + failure logging,
   bounded `ipcPerfLog`), mock fallback for non-Tauri contexts, and
   command-name type safety via the `CommandName` union.
4. **Contract tests** enforce drift-freeness on every CI run:
   - `packages/client/src/__tests__/commandContract.test.ts` asserts the
     hand-maintained `IPC_COMMANDS` list matches
     `generated/ipc-commands.json` exactly. A mismatch is accompanied
     by step-by-step remediation instructions in the assertion message.
   - `packages/client/src/__tests__/generated.drift.test.ts` compares
     generated shapes against the hand-written mirrors in
     `@tracepilot/types` (Wave 21).
   - Rust-side, `ipc_manifest_tests` in `tracepilot-tauri-bindings`
     asserts the manifest matches the registered command set.
5. **Wire-format preservation.** `SessionEvent.data` is serialised as
   `Record<string, unknown>` because downstream CLI schema changes
   happen outside our control. The `narrowSessionEvent` helper in
   `@tracepilot/types` (`session-event-payloads.ts`) performs a
   tagged-union narrowing at the UI layer — the wire format stays
   loose, the consumer sees a discriminated union. See
   `apps/desktop/src/utils/orchestratorActivity.ts` for usage.

## Consequences

- **Positive**: A Rust-side rename is caught by `pnpm gen:bindings` +
  TypeScript compile in a single PR — no runtime surprise.
- **Positive**: Command names are typo-proof on both sides — the
  contract test prevents the TypeScript `IPC_COMMANDS` list from
  drifting from Rust.
- **Positive**: Adding a new command is a mechanical three-step
  checklist documented in the test's failure message.
- **Negative**: Every IPC-exposed Rust type must derive `specta::Type`
  and cannot use types Specta can't serialise (e.g. opaque handles).
  Workarounds use newtypes.
- **Negative**: `bindings.ts` shows up in PR diffs even for small Rust
  changes. Worth the signal.

## Alternatives considered

- **Keep hand-written types**. Rejected — Wave 21 cleanup found dozens
  of drift bugs that had survived for months.
- **Generate into `@tracepilot/types` directly**. Rejected — we want
  the generated file to be a tight command surface, while
  `@tracepilot/types` also exports hand-authored helpers
  (`narrowSessionEvent`) and contract-test mirrors.
- **Typed wire format for `SessionEvent.data`**. Rejected — event
  shapes come from the Copilot CLI which we do not control; forcing a
  closed enum would silently drop unknown events. The narrower pattern
  gives us type safety at the call site without discarding data.
