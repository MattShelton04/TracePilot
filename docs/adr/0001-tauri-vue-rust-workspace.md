# ADR-0001: Tauri 2 + Vue 3 + Rust workspace architecture

Date: 2026-04-22
Status: Accepted

## Context

TracePilot is a desktop analytics tool for the GitHub Copilot CLI. It must:

- Render a rich, reactive UI with hundreds of session rows, live streaming
  events, and virtualised tables.
- Run long-lived background processes (bridge, indexer, MCP host, skill
  runners) with tight control over child-process lifecycle, stdout capture
  and Windows console handling.
- Read large SQLite databases and stream JSONL event files from disk.
- Ship as a signed, auto-updating desktop binary on Windows first, macOS
  and Linux as tier-2 (see ADR-0010).

An Electron-style "bundle a browser + Node" stack was rejected early:
session files can be tens of MB, the UI needs to stay at 60fps while the
backend streams events, and we want native OS integration (notifications,
taskbar progress, single-instance) without the 150 MB memory floor.
Neutralino/Wails were considered but lacked the Rust ecosystem we needed
for parsing, SQLite access, and the MCP/skill orchestration work that
dominates the backend.

## Decision

TracePilot is structured as:

1. **Tauri 2 shell** (`apps/desktop/src-tauri`) — a Rust binary that owns
   the webview window, the IPC surface (plugin commands + events) and the
   lifecycle of every background worker. Tauri 2 was chosen over Tauri 1
   for the capability-based permission model (see ADR-0011) and the
   stabilised plugin system.
2. **Vue 3 + Composition API front-end** (`apps/desktop/src`) — single
   page app built with Vite, `vue-tsc` for type-checking, Pinia stores
   for state (see ADR-0006). Composition API was chosen over Options API
   so state logic composes cleanly across stores and so `script setup`
   SFCs stay small enough for the file-size gate.
3. **Rust Cargo workspace** (`Cargo.toml` at the repo root) with
   feature-aligned crates:
   - `tracepilot-core` — domain models, parsing, SQLite helpers, ids,
     errors.
   - `tracepilot-indexer` — session indexing + Tantivy search index.
   - `tracepilot-orchestrator` — long-lived processes, bridge, MCP host,
     skill runners.
   - `tracepilot-export` — report + artefact generation.
   - `tracepilot-tauri-bindings` — the Tauri plugin that exposes IPC
     commands and events to the webview.
   - `tracepilot-bench`, `tracepilot-test-support` — dev-only.
4. **pnpm workspace** (`pnpm-workspace.yaml`) with three reusable
   packages: `@tracepilot/client` (IPC wrappers + generated bindings),
   `@tracepilot/types` (shared type mirrors + event narrowers),
   `@tracepilot/ui` (Vue components + `runAction` / `runMutation`).

The workspace layout keeps a single `Cargo.lock` and a single
`pnpm-lock.yaml`, and shared dependency versions are pinned via
`[workspace.dependencies]` (Rust) and the pnpm catalog
(`pnpm-workspace.yaml`, see Wave 104).

## Consequences

- **Positive**: one Rust compile tree means `cargo check` catches
  cross-crate breaks immediately; shared dependency versions prevent
  duplicate builds of `serde`/`tokio`; Vue 3 Composition API + Pinia
  scales to the ~70 stores/composables we have today.
- **Positive**: the Tauri plugin model forces every IPC command through
  one registration point (`tracepilot-tauri-bindings`), which is what
  makes the Specta contract (ADR-0002) and the capability scoping
  (ADR-0011) possible.
- **Negative**: contributors need both a Rust toolchain and a Node/pnpm
  toolchain. Mitigated by `justfile` (Wave 107) and `copilot-setup-
  steps.yml`.
- **Negative**: Tauri's Rust version floor drags our MSRV. We pin to
  `edition = "2024"` in the workspace and accept that as the cost.

## Alternatives considered

- **Electron + TypeScript backend**. Rejected: 150 MB memory floor,
  Node process model is a poor fit for our long-lived child-process
  orchestration, no ergonomic access to Rust crates we already wanted
  (Tantivy, `rusqlite`, `mcp` client libraries).
- **Neutralino / Wails**. Rejected: smaller ecosystems, fewer plugins
  (updater, notifications, single-instance), less mature Windows story.
- **Separate frontend + backend repos**. Rejected: the IPC contract
  (ADR-0002) is generated by a Rust binary that writes into
  `packages/client/src/generated/` — co-located repos keep that loop
  atomic and let a single PR land both sides of a breaking change.
- **Vue 3 Options API**. Rejected: file-size gate + composability. The
  Pinia convention in ADR-0006 depends on Composition-API idioms.
