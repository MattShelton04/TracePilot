# @tracepilot/types

Shared TypeScript types and API contracts for TracePilot. Pure types plus a
handful of constants and tiny helpers — no runtime I/O, no framework code.

Types that mirror a Rust DTO live here when they're hand-written; DTOs
generated by specta from `tracepilot-tauri-bindings` live in
`@tracepilot/client/src/generated/bindings.ts` instead (see ADR
[0002 — IPC contract via specta-generated bindings](../../docs/adr/0002-ipc-contract-specta-bindings.md)).

## Public API

Everything is re-exported from `src/index.ts`. Module groups:

| Module                     | Key exports                                                  |
| -------------------------- | ------------------------------------------------------------ |
| `session`                  | `SessionListItem`, `SessionDetail`, `SessionHealth`, todos   |
| `conversation`             | `ConversationTurn`, `AttributedMessage`, `SessionEvent`      |
| `session-event-payloads`   | Event-type unions + `narrowSessionEvent` narrower            |
| `search`                   | `SearchResult`, `SearchFilters`, `SearchFacetsResponse`      |
| `analytics`                | `AnalyticsData`, `ProductivityMetrics`, `ToolUsageEntry`     |
| `export`                   | `ExportFormat`, `ExportConfig`, `SectionId`, preview/result  |
| `orchestration`            | `WorktreeInfo`, `LaunchedSession`, `RegisteredRepo`, configs |
| `tasks`                    | `Task`, `TaskStatus`, `OrchestratorHealth`, presets          |
| `mcp`                      | `McpServerConfig`, `McpHealthResult`, diff types             |
| `skills`                   | `Skill`, `SkillSummary`, import previews                     |
| `models`                   | `MODEL_REGISTRY`, `ModelDefinition`, tier helpers            |
| `config` + `defaults`      | `TracePilotConfig`, `createDefaultConfig`, all defaults      |
| `known-events`             | `TRACEPILOT_KNOWN_EVENTS` (wire-name allow-list)             |
| `tool-args`, `tool-rendering` | Tool arg helpers + per-tool render preferences             |
| `utils/formatters`         | `formatBytes`, `formatDuration`, `formatCost`, …             |

## Usage

```ts
import type { SessionListItem, SessionDetail } from "@tracepilot/types";
import { formatDuration, isTerminalStatus } from "@tracepilot/types";
```

## Workspace dependencies

None. This package is intentionally dependency-free so it can be consumed from
any other workspace package without creating cycles.

## Layout

- `src/index.ts` — public barrel. Additions must keep grouping comments in sync.
- `src/<domain>.ts` — one file per functional area.
- `src/utils/formatters.ts` — the only runtime code; pure formatting helpers.
- `src/defaults.ts` — configuration defaults and `createDefaultConfig()`.
- `src/models.ts` — the `MODEL_REGISTRY` pricing / tier source of truth.
