## 2024-05-22 - Fix path traversal vulnerability in imports
**Vulnerability:** Found a cross-platform path traversal vulnerability in `crates/tracepilot-orchestrator/src/skills/import/github.rs` and `crates/tracepilot-export/src/import/validator/safety.rs` where path checks using `std::path::Component::ParentDir` or `is_absolute()` do not adequately catch backslash-based paths (`\`) or root directory characters on Unix systems when processing potentially malicious Windows paths. Additionally, naive string matching `s.contains("..")` was being used, which could cause functional regressions for perfectly safe files like `my..file.txt`.
**Learning:** Unix systems do not recognize Windows backslash separators in paths, meaning that path traversal attempts using `\` might bypass `Component` analysis and `is_absolute` checks on Unix systems. Naive string matching is also a poor choice for path validation.
**Prevention:** Always explicitly check for backslashes `\` and explicit root characters `/` when validating cross-platform paths, and avoid naive string matching like `.contains("..")`, continuing to rely on `std::path::Component::ParentDir` alongside the explicit character checks.
