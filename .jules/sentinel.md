## 2026-05-18 - Prevent Path Traversal in GitHub Asset Imports and Filename Validation
**Vulnerability:** Path traversal vulnerability due to insufficient validation in GitHub skill asset imports (`crates/tracepilot-orchestrator/src/skills/import/github.rs`) and general filename validation (`crates/tracepilot-orchestrator/src/validation.rs` and `crates/tracepilot-export/src/import/validator/safety.rs`). The code incorrectly relied only on `std::path::Component::ParentDir` (`..`) and `is_absolute()`.
**Learning:** Unix systems do not recognize Windows backslash (`\`) separators or Windows absolute paths, so `Path::is_absolute()` is insufficient for cross-platform validation. Explicitly checking for backslashes (`\`), forward slashes (`/`), and `..` is required. Naive string matching like `contains("..")` can cause false positives (e.g., `file..name`), so using `Component::ParentDir` is still necessary alongside character checks where applicable.
**Prevention:** Always explicitly check for both `\` and `/` when validating paths from external sources (like GitHub trees or user inputs) to prevent bypassing directory restrictions, regardless of the host OS.
