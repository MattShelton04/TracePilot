## 2024-05-17 - Cross-platform path validation bypass via OS-specific path parsing
**Vulnerability:** Path validation relying solely on `std::path::Component` and `is_absolute()` can be bypassed because the `Path` API parses paths according to the host OS semantics. On Unix systems, Windows backslashes (`\`) and absolute paths (`C:\`) are treated as valid filename characters rather than path separators, allowing traversal sequences to slip through. Additionally, naive string matching like `.contains("..")` was used as a fallback but causes functional regressions by falsely flagging legitimate files like `my..file.txt`.
**Learning:** Standard library path APIs are OS-dependent and insufficient for validating untrusted cross-platform archive or git tree paths (which may originate from a different OS).
**Prevention:** Always combine `std::path::Component::ParentDir` checks with explicit substring blocks for backslashes (`\`) and root characters (`/`) when processing cross-platform paths. Never use `.contains("..")` for path validation.
