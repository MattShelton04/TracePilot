## 2026-03-10 - Pre-allocate large SQL placeholder strings for memory optimization
**Learning:** In Rust when dynamically building SQL placeholders `?, ?, ?` and batched values `(?), (?)` for bulk inserts or IN clauses, using intermediate strings and concatenations like `push()` repeatedly is slow. We should pre-allocate the final required string length exactly and use `.push_str()` instead of `.push()` where feasible, as `push_str` maps to a direct memory copy that bypasses scalar bound checks. Crucially, calculating the allocation capacity should avoid bounds checking math (`saturating_add/sub/mul`) which slows execution.
**Action:** Always prefer `String::with_capacity(exact_len)` with standard math operators (+, -, *) for SQL statement construction. If prepending a single char to enable chunking, add a `n==0` short-circuit guard instead of trying to manipulate bounds checking.
