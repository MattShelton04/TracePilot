## 2024-05-12 - [Rust SQL String Interpolation Optimization]
**Learning:** Appending static strings with `push_str()` provides a performance improvement compared to iteratively chaining scalar `.push(char)` loops for string interpolation. `push_str()` natively utilizes memory-slice copying without the validation overhead of pushing scalar characters. However, care must be taken to maintain the default/standard math when generating capacities dynamically instead of relying on `saturating_` unsigned math because they often de-optimize due to bound-check instructions generated by the compiler.
**Action:** When dynamically generating large SQL placeholder strings in Rust for batch inserts or clauses, append slices (`push_str(", ?")`) rather than looping single-character pushes to maximize memory write speed.
