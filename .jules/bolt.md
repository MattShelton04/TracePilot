## 2024-05-20 - Optimize dynamic SQL placeholder string generation in Rust
**Learning:** Using scalar character-by-character pushes (`.push(',')`, `.push('?')`) in a loop to construct SQL placeholders introduces performance overhead. Utilizing `.push_str()` for static string chunks (`.push_str(",?")`) performs a direct, optimized memory copy of the pre-allocated byte slice and avoids iterative single-character appends. For generating large sequences like `?, ?, ?`, it is faster to extract the first character push before the loop and append the rest as identical string chunks (`", ?"`), while checking edge cases like `n=0`.
**Action:** Always prefer pre-allocating an exact buffer capacity and appending static substring chunks using `.push_str()` over individual `.push()` calls for repetitive pattern generation. Remember to explicitly short-circuit edge cases like `n=0` to avoid runtime panics or incorrectly structured output when extracting the initial element before chunking loops.
