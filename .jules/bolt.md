## 2024-05-15 - [Optimize SQL Placeholder String Generation]
**Learning:** When generating dynamic SQL placeholders in Rust, using scalar character-by-character pushes (`.push(',')`, `.push('?')`) is less efficient than using chunk slice additions (`push_str`) directly onto the pre-allocated string. `push_str` performs a direct, optimized memory copy of the pre-allocated byte slice. Avoid using `.map()`, `format!()`, and `.join()` as they cause excessive intermediate memory allocations. Avoid replacing standard arithmetic operators with `.saturating_add()`, `.saturating_sub()`, or `.saturating_mul()` when calculating capacity, as it forces the compiler to inject bounds-checking instructions.
**Action:** Use `.push_str()` (e.g., `.push_str(", ?")`) for appending string slices when building dynamic SQL queries.
