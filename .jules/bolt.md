## 2024-05-13 - String Pre-allocation and Push Methods
**Learning:** When generating dynamic SQL placeholders in Rust, do not blindly remove an intermediate string allocation that pre-builds a repeating chunk (like a single row representation e.g. `(?, ?, ?)`) outside of a loop. Pre-building a chunk once and appending that exact string multiple times using `.push_str()` is an O(num_rows) operation and significantly outperforms iteratively re-generating the chunk characters inside the loop, even if it requires a single intermediate string allocation. Also, `push_str(", ?")` performs a direct memory copy and is faster than `.push(',')` and `.push('?')`. Calculate capacity strictly using arithmetic.
**Action:** Use `.push_str()` for chunks and exact arithmetic capacity calculation.
