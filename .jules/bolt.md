## 2024-05-18 - SQLite batched insert optimization
**Learning:** `batched_insert` creates long parameterized SQL strings using chunks of 50 in `tracepilot-indexer`. However, `rusqlite` has a `json_each` extension which is highly optimized for bulk inserts by parsing JSON strings without hitting the `SQLITE_MAX_VARIABLE_NUMBER` limit. Memory file also stated: "When performing bulk inserts or handling dynamically sized arrays in SQLite with `rusqlite` in Rust, an optimal pattern is to serialize the Rust collection into a JSON array (e.g., via `serde_json::to_string`) and execute a single statement using the `json_each(?)` extension (e.g., `INSERT INTO ... SELECT value FROM json_each(?)`). This bypasses the `SQLITE_MAX_VARIABLE_NUMBER` parameter limit, eliminates loop execution overhead, and avoids complex chunking logic."
**Action:** Replace `batched_insert` with `json_each` inserts where applicable to improve SQLite insertion performance, specifically in `crates/tracepilot-indexer/src/index_db/batch_insert.rs` or directly in `session_writer.rs` and `search_writer/mod.rs`
