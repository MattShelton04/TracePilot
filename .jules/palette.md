## 2024-05-02 - Ensure Interactive Icon-only Buttons Have Dynamic ARIA Labels
**Learning:** Found an accessibility issue pattern where icon-only buttons with dynamic tooltips (like "Copied!" vs "Copy content") lacked dynamic `aria-label` attributes to maintain parity with visual changes. Purely decorative SVGs inside these buttons were also missing `aria-hidden="true"`, causing screen readers to potentially announce raw svg descriptions rather than clear actions.
**Action:** Always verify that interactive icon-only elements (like copy buttons) use dynamic `aria-label` attributes reflecting their current state (e.g., `:aria-label="copied ? 'Copied!' : 'Copy content'"`) and ensure decorative child SVGs have `aria-hidden="true"` so screen readers properly announce the intended state change.
